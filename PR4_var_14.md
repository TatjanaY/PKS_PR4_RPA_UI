# ✅ Вариант 14

## 🟢 Задание 1 (базовый уровень)

### Постановка задачи

Создайте пользовательскую форму **«Учёт участника хакатона»** для последовательного ввода данных нескольких разработчиков.

### Элементы формы

Добавьте:

1.  Текст (строка) — Никнейм участника (`participant_nickname`)
2.  Текст (строка) — Email для связи (`contact_email`)
3.  Число — Опыт программирования в годах (`coding_experience`, min = 0, max = 20)
4.  Кнопки:

   *   Подтвердить (ОК)
   *   Отмена

---

### Подготовка данных

С помощью любого ИИ-сервиса (DeepSeek / ChatGPT) сгенерируйте **10 тестовых записей** вида:

```
Никнейм, Email, Опыт_лет
```

---

### Требования к алгоритму

1.  Форма вызывается **в цикле 10 раз**.
2.  На каждой итерации вводится один набор данных.
3.  После нажатия ОК данные добавляются в файл `hackathon_participants.txt`.
4.  Каждая запись сохраняется с новой строки в формате:

   ```
   Никнейм;Email;Опыт
   ```
5.  При нажатии Отмена ввод прекращается.
6.  После завершения вывести сообщение:

   ```
   Зарегистрировано участников: N
   ```

# 🔵 Задание 2 (повышенный уровень)

### Постановка задачи

Создайте форму **«Расширенный учёт участника хакатона»** с дополнительными элементами, условиями показа и использованием кнопки типа **Триггер**.

---

### Элементы формы

1.  Текст — Никнейм участника (`participant_nickname`)
2.  Текст — Email для связи (`contact_email`)
3.  Число — Опыт программирования (`coding_experience`)
4.  Чекбокс — Есть команда (`has_team`)
5.  Текст — Название команды (`team_name`)

   *   отображать только если `has_team = Истина`
6.  Селект — Трек участия (`hackathon_track`)

   *   варианты: ai, web, mobile
7.  Кнопки:

   *   Триггер **«Проверить допуск»**
   *   ОК
   *   Отмена

---

### Логика кнопки Триггер

Создайте функцию `check_hackathon_eligibility(form_result)`, которая:

1.  Проверяет:

   *   `contact_email` содержит домен `@gmail.com` ИЛИ `@yandex.ru`
   *   И (`coding_experience` >= 1 ИЛИ `has_team` = Истина)
2.  Формирует текст:

   *   «Участник допущен к хакатону»
   *   или «Ошибка: неверный email или недостаточный опыт без команды»

Результатом работы функции является вывод сообщения на экран

⚠ Форма не должна закрываться при нажатии Триггера.

---

### Динамические параметры при вызове формы

Программно задать:

*   заголовок окна: **Hackathon 2026 💻**
*   изменить цвет фона;
*   значение по умолчанию: `hackathon_track = web`.

---

### Подготовка данных

Сгенерируйте с помощью ИИ **10 расширенных записей**:

```
Никнейм, Email, Опыт, Команда (да/нет), Название_команды, Трек
```

---

### Требования к алгоритму

1.  Ввод выполняется последовательно для 10 участников.
2.  Перед подтверждением обязательно нажимать кнопку «Проверить допуск».
3.  После нажатия ОК данные сохраняются в файл `hackathon_participants_extended.csv`:

```
Никнейм;Email;Опыт;Команда;Название_команды;Трек
```

4.  Если форма возвращает пустое значение (Отмена), выполнение прекращается.
