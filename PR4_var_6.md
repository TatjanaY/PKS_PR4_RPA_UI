# ✅ Вариант 6

## 🟢 Задание 1 (базовый уровень)

### Постановка задачи

Создайте пользовательскую форму **«Учёт абонемента в бассейн»** для последовательного ввода данных нескольких посетителей.

### Элементы формы

Добавьте:

1.  Текст (строка) — ФИО пловца (`swimmer_name`)
2.  Текст (строка) — Контактный телефон (`contact_phone`)
3.  Число — Количество занятий (`lesson_count`, min = 1, max = 100)
4.  Кнопки:

   *   Подтвердить (ОК)
   *   Отмена

---

### Подготовка данных

С помощью любого ИИ-сервиса (DeepSeek / ChatGPT) сгенерируйте **10 тестовых записей** вида:

```
ФИО, Телефон, Количество_занятий
```

---

### Требования к алгоритму

1.  Форма вызывается **в цикле 10 раз**.
2.  На каждой итерации вводится один набор данных.
3.  После нажатия ОК данные добавляются в файл `pool_members.txt`.
4.  Каждая запись сохраняется с новой строки в формате:

   ```
   ФИО;Телефон;Занятия
   ```
5.  При нажатии Отмена ввод прекращается.
6.  После завершения вывести сообщение:

   ```
   Оформлено абонементов: N
   ```

# 🔵 Задание 2 (повышенный уровень)

### Постановка задачи

Создайте форму **«Расширенный учёт абонемента»** с дополнительными элементами, условиями показа и использованием кнопки типа **Триггер**.

---

### Элементы формы

1.  Текст — ФИО пловца (`swimmer_name`)
2.  Текст — Контактный телефон (`contact_phone`)
3.  Число — Возраст (`swim_age`)
4.  Чекбокс — Нужен шкафчик (`has_locker`)
5.  Текст — Номер шкафчика (`locker_number`)

   *   отображать только если `has_locker = Истина`
6.  Селект — Тип абонемента (`membership_plan`)

   *   варианты: month, quarter, year
7.  Кнопки:

   *   Триггер **«Проверить абонемент»**
   *   ОК
   *   Отмена

---

### Логика кнопки Триггер

Создайте функцию `check_membership(form_result)`, которая:

1.  Проверяет:

   *   `contact_phone` состоит только из цифр
   *   И (`swim_age` >= 10 ИЛИ `has_locker` = Истина)
2.  Формирует текст:

   *   «Абонемент действителен»
   *   или «Ошибка: неверный телефон или возраст без шкафчика»

Результатом работы функции является вывод сообщения на экран

⚠ Форма не должна закрываться при нажатии Триггера.

---

### Динамические параметры при вызове формы

Программно задать:

*   заголовок окна: **Бассейн «Волна» 🏊**
*   изменить цвет фона;
*   значение по умолчанию: `membership_plan = month`.

---

### Подготовка данных

Сгенерируйте с помощью ИИ **10 расширенных записей**:

```
ФИО, Телефон, Возраст, Шкафчик (да/нет), Номер_шкафчика, Тип_абонемента
```

---

### Требования к алгоритму

1.  Ввод выполняется последовательно для 10 посетителей.
2.  Перед подтверждением обязательно нажимать кнопку «Проверить абонемент».
3.  После нажатия ОК данные сохраняются в файл `pool_members_extended.csv`:

```
ФИО;Телефон;Возраст;Шкафчик;Номер_шкафчика;Тип_абонемента
```

4.  Если форма возвращает пустое значение (Отмена), выполнение прекращается.
