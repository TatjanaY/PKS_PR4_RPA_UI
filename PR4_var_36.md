# ✅ Вариант 36

## 🟢 Задание 1 (базовый уровень)

### Постановка задачи

Создайте пользовательскую форму **«Учёт посетителя кулинарного курса»** для последовательного ввода данных нескольких участников мастер-класса.

### Элементы формы

Добавьте:

1.  Текст (строка) — ФИО участника (`chef_participant`)
2.  Текст (строка) — Контактный телефон (`contact_phone`)
3.  Число — Уровень кулинарного опыта (1–10) (`cooking_skill`, min = 1, max = 10)
4.  Кнопки:

   *   Подтвердить (ОК)
   *   Отмена

---

### Подготовка данных

С помощью любого ИИ-сервиса (DeepSeek / ChatGPT) сгенерируйте **10 тестовых записей** вида:

```
ФИО, Телефон, Уровень_опыта
```

---

### Требования к алгоритму

1.  Форма вызывается **в цикле 10 раз**.
2.  На каждой итерации вводится один набор данных.
3.  После нажатия ОК данные добавляются в файл `cooking_course.txt`.
4.  Каждая запись сохраняется с новой строки в формате:

   ```
   ФИО;Телефон;Уровень_опыта
   ```
5.  При нажатии Отмена ввод прекращается.
6.  После завершения вывести сообщение:

   ```
   Зарегистрировано участников: N
   ```

# 🔵 Задание 2 (повышенный уровень)

### Постановка задачи

Создайте форму **«Расширенный учёт кулинарного курса»** с дополнительными элементами, условиями показа и использованием кнопки типа **Триггер**.

---

### Элементы формы

1.  Текст — ФИО участника (`chef_participant`)
2.  Текст — Контактный телефон (`contact_phone`)
3.  Число — Уровень кулинарного опыта (`cooking_skill`)
4.  Чекбокс — Нужен персональный фартук (`needs_apron`)
5.  Текст — Размер фартука (`apron_size`)

   *   отображать только если `needs_apron = Истина`
6.  Селект — Выбранное блюдо (`selected_dish`)

   *   варианты: pasta, sushi, dessert
7.  Кнопки:

   *   Триггер **«Проверить запись»**
   *   ОК
   *   Отмена

---

### Логика кнопки Триггер

Создайте функцию `validate_cooking_enrollment(form_result)`, которая:

1.  Проверяет:

   *   `contact_phone` содержит только цифры И длина >= 10
   *   И (`cooking_skill` >= 4 ИЛИ `selected_dish` == "pasta")
2.  Формирует текст:

   *   «Участник допущен к курсу»
   *   или «Ошибка: неверный телефон или недостаточный опыт для выбранного блюда»

Результатом работы функции является вывод сообщения на экран

⚠ Форма не должна закрываться при нажатии Триггера.

---

### Динамические параметры при вызове формы

Программно задать:

*   заголовок окна: **Кулинарная Школа 👨‍🍳**
*   изменить цвет фона;
*   значение по умолчанию: `selected_dish = pasta`.

---

### Подготовка данных

Сгенерируйте с помощью ИИ **10 расширенных записей**:

```
ФИО, Телефон, Уровень_опыта, Фартук (да/нет), Размер_фартука, Блюдо
```

---

### Требования к алгоритму

1.  Ввод выполняется последовательно для 10 участников.
2.  Перед подтверждением обязательно нажимать кнопку «Проверить запись».
3.  После нажатия ОК данные сохраняются в файл `cooking_course_extended.csv`:

```
ФИО;Телефон;Уровень_опыта;Фартук;Размер_фартука;Блюдо
```

4.  Если форма возвращает пустое значение (Отмена), выполнение прекращается.
