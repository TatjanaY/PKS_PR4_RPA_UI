# ✅ Вариант 7

## 🟢 Задание 1 (базовый уровень)

### Постановка задачи

Создайте пользовательскую форму **«Регистрация гостя отеля»** для последовательного ввода данных нескольких постояльцев.

### Элементы формы

Добавьте:

1.  Текст (строка) — ФИО (`guest_name`)
2.  Текст (строка) — Серия паспорта (`passport_series`)
3.  Число — Количество ночей (`nights_count`, min = 1, max = 30)
4.  Кнопки:

   *   Подтвердить (ОК)
   *   Отмена

---

### Подготовка данных

С помощью любого ИИ-сервиса (DeepSeek / ChatGPT) сгенерируйте **10 тестовых записей** вида:

```
ФИО, Серия_паспорта, Количество_ночей
```

---

### Требования к алгоритму

1.  Форма вызывается **в цикле 10 раз**.
2.  На каждой итерации вводится один набор данных.
3.  После нажатия ОК данные добавляются в файл `hotel_guests.txt`.
4.  Каждая запись сохраняется с новой строки в формате:

   ```
   ФИО;Серия_паспорта;Количество_ночей
   ```
5.  При нажатии Отмена ввод прекращается.
6.  После завершения вывести сообщение:

   ```
   Зарегистрировано гостей: N
   ```

# 🔵 Задание 2 (повышенный уровень)

### Постановка задачи

Создайте форму **«Расширенная регистрация отеля»** с дополнительными элементами, условиями показа и использованием кнопки типа **Триггер**.

---

### Элементы формы

1.  Текст — ФИО (`guest_name`)
2.  Текст — Серия паспорта (`passport_series`)
3.  Число — Количество ночей (`nights_count`)
4.  Чекбокс — VIP статус (`is_vip`)
5.  Текст — Номер карты лояльности (`loyalty_card`)

   *   отображать только если `is_vip = Истина`
6.  Селект — Тип номера (`room_type`)

   *   варианты: standard, deluxe, suite
7.  Кнопки:

   *   Триггер **«Проверить бронь»**
   *   ОК
   *   Отмена

---

### Логика кнопки Триггер

Создайте функцию `validate_booking(form_result)`, которая:

1.  Проверяет:

   *   (Длина серии паспорта == 4 И серия состоит только из цифр)
   *   И (Количество ночей <= 14 ИЛИ VIP статус = Истина)
2.  Формирует текст:

   *   «Бронь подтверждена»
   *   или «Ошибка: неверный паспорт или превышен лимит дней без VIP»

Результатом работы функции является вывод сообщения на экран

⚠ Форма не должна закрываться при нажатии Триггера.

---

### Динамические параметры при вызове формы

Программно задать:

*   заголовок окна: **Hotel Resort 🏨**
*   изменить цвет фона;
*   значение по умолчанию: `room_type = standard`.

---

### Подготовка данных

Сгенерируйте с помощью ИИ **10 расширенных записей**:

```
ФИО, Серия_паспорта, Количество_ночей, VIP (да/нет), Карта_лояльности, Тип_номера
```

---

### Требования к алгоритму

1.  Ввод выполняется последовательно для 10 гостей.
2.  Перед подтверждением обязательно нажимать кнопку «Проверить бронь».
3.  После нажатия ОК данные сохраняются в файл `hotel_guests_extended.csv`:

```
ФИО;Серия_паспорта;Количество_ночей;VIP;Карта_лояльности;Тип_номера
```

4.  Если форма возвращает пустое значение (Отмена), выполнение прекращается.
