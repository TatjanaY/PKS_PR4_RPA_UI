# ✅ Вариант 13

## 🟢 Задание 1 (базовый уровень)

### Постановка задачи

Создайте пользовательскую форму **«Регистрация донора крови»** для последовательного ввода данных нескольких потенциальных доноров.

### Элементы формы

Добавьте:

1.  Текст (строка) — ФИО донора (`donor_fullname`)
2.  Текст (строка) — Номер полиса (`policy_number`)
3.  Число — Вес в кг (`weight_kg`, min = 50, max = 150)
4.  Кнопки:

   *   Подтвердить (ОК)
   *   Отмена

---

### Подготовка данных

С помощью любого ИИ-сервиса (DeepSeek / ChatGPT) сгенерируйте **10 тестовых записей** вида:

```
ФИО, Номер_полиса, Вес_кг
```

---

### Требования к алгоритму

1.  Форма вызывается **в цикле 10 раз**.
2.  На каждой итерации вводится один набор данных.
3.  После нажатия ОК данные добавляются в файл `blood_donors.txt`.
4.  Каждая запись сохраняется с новой строки в формате:

   ```
   ФИО;Номер_полиса;Вес_кг
   ```
5.  При нажатии Отмена ввод прекращается.
6.  После завершения вывести сообщение:

   ```
   Зарегистрировано доноров: N
   ```

# 🔵 Задание 2 (повышенный уровень)

### Постановка задачи

Создайте форму **«Расширенная регистрация донора»** с дополнительными элементами, условиями показа и использованием кнопки типа **Триггер**.

---

### Элементы формы

1.  Текст — ФИО донора (`donor_fullname`)
2.  Текст — Номер полиса (`policy_number`)
3.  Число — Вес в кг (`weight_kg`)
4.  Чекбокс — Первая сдача крови (`is_first_donation`)
5.  Текст — Дата последней мед. проверки (`last_checkup_date`)

   *   отображать только если `is_first_donation = Ложь`
6.  Селект — Группа крови (`blood_group`)

   *   варианты: O, A, B, AB
7.  Кнопки:

   *   Триггер **«Пройти скрининг»**
   *   ОК
   *   Отмена

---

### Логика кнопки Триггер

Создайте функцию `run_donor_screening(form_result)`, которая:

1.  Проверяет:

   *   (Вес >= 55 И длина номера полиса == 11)
   *   ИЛИ (Первая сдача = Истина И Группа крови в ["O", "A"])
2.  Формирует текст:

   *   «Скрининг пройден»
   *   или «Ошибка: вес/полис не соответствуют или группа не подходит для первой сдачи»

Результатом работы функции является вывод сообщения на экран

⚠ Форма не должна закрываться при нажатии Триггера.

---

### Динамические параметры при вызове формы

Программно задать:

*   заголовок окна: **Blood Donor Center 🩸**
*   изменить цвет фона;
*   значение по умолчанию: `blood_group = O`.

---

### Подготовка данных

Сгенерируйте с помощью ИИ **10 расширенных записей**:

```
ФИО, Номер_полиса, Вес_кг, Первая_сдача (да/нет), Дата_проверки, Группа_крови
```

---

### Требования к алгоритму

1.  Ввод выполняется последовательно для 10 доноров.
2.  Перед подтверждением обязательно нажимать кнопку «Пройти скрининг».
3.  После нажатия ОК данные сохраняются в файл `blood_donors_extended.csv`:

```
ФИО;Номер_полиса;Вес_кг;Первая_сдача;Дата_проверки;Группа_крови
```

4.  Если форма возвращает пустое значение (Отмена), выполнение прекращается.
