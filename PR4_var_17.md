# ✅ Вариант 17

## 🟢 Задание 1 (базовый уровень)

### Постановка задачи

Создайте пользовательскую форму **«Регистрация клиента автосервиса»** для последовательного ввода данных нескольких владельцев автомобилей.

### Элементы формы

Добавьте:

1.  Текст (строка) — ФИО владельца (`owner_name`)
2.  Текст (строка) — Марка автомобиля (`car_brand`)
3.  Число — Год выпуска авто (`car_year`, min = 1990, max = 2024)
4.  Кнопки:

   *   Подтвердить (ОК)
   *   Отмена

---

### Подготовка данных

С помощью любого ИИ-сервиса (DeepSeek / ChatGPT) сгенерируйте **10 тестовых записей** вида:

```
ФИО, Марка_авто, Год_выпуска
```

---

### Требования к алгоритму

1.  Форма вызывается **в цикле 10 раз**.
2.  На каждой итерации вводится один набор данных.
3.  После нажатия ОК данные добавляются в файл `auto_clients.txt`.
4.  Каждая запись сохраняется с новой строки в формате:

   ```
   ФИО;Марка_авто;Год_выпуска
   ```
5.  При нажатии Отмена ввод прекращается.
6.  После завершения вывести сообщение:

   ```
   Зарегистрировано клиентов: N
   ```

# 🔵 Задание 2 (повышенный уровень)

### Постановка задачи

Создайте форму **«Расширенная регистрация автосервиса»** с дополнительными элементами, условиями показа и использованием кнопки типа **Триггер**.

---

### Элементы формы

1.  Текст — ФИО владельца (`owner_name`)
2.  Текст — Марка автомобиля (`car_brand`)
3.  Число — Год выпуска авто (`car_year`)
4.  Чекбокс — Гарантийное обслуживание (`is_warranty`)
5.  Текст — Номер сервисной книжки (`service_book_num`)

   *   отображать только если `is_warranty = Истина`
6.  Селект — Тип работ (`service_type`)

   *   варианты: diagnostic, repair, maintenance
7.  Кнопки:

   *   Триггер **«Проверить приём»**
   *   ОК
   *   Отмена

---

### Логика кнопки Триггер

Создайте функцию `validate_service_acceptance(form_result)`, которая:

1.  Проверяет:

   *   (Год выпуска >= 2015 И длина марки авто >= 3)
   *   ИЛИ (Гарантийное обслуживание = Истина И длина номера сервисной книжки == 10)
2.  Формирует текст:

   *   «Автомобиль принят в сервис»
   *   или «Ошибка: авто старше 2015 или неверный номер книжки для гарантии»

Результатом работы функции является вывод сообщения на экран

⚠ Форма не должна закрываться при нажатии Триггера.

---

### Динамические параметры при вызове формы

Программно задать:

*   заголовок окна: **Auto Service 🚗**
*   изменить цвет фона;
*   значение по умолчанию: `service_type = maintenance`.

---

### Подготовка данных

Сгенерируйте с помощью ИИ **10 расширенных записей**:

```
ФИО, Марка_авто, Год_выпуска, Гарантия (да/нет), Номер_книжки, Тип_работ
```

---

### Требования к алгоритму

1.  Ввод выполняется последовательно для 10 клиентов.
2.  Перед подтверждением обязательно нажимать кнопку «Проверить приём».
3.  После нажатия ОК данные сохраняются в файл `auto_clients_extended.csv`:

```
ФИО;Марка_авто;Год_выпуска;Гарантия;Номер_книжки;Тип_работ
```

4.  Если форма возвращает пустое значение (Отмена), выполнение прекращается.
