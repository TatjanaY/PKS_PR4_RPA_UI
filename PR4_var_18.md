# ✅ Вариант 18

## 🟢 Задание 1 (базовый уровень)

### Постановка задачи

Создайте пользовательскую форму **«Заявка на ремонт техники»** для последовательного ввода данных нескольких обращений клиентов.

### Элементы формы

Добавьте:

1.  Текст (строка) — ФИО владельца (`owner_fullname`)
2.  Текст (строка) — Модель устройства (`device_model`)
3.  Число — Стоимость ремонта в рублях (`repair_cost`, min = 500, max = 50000)
4.  Кнопки:

   *   Подтвердить (ОК)
   *   Отмена

---

### Подготовка данных

С помощью любого ИИ-сервиса (DeepSeek / ChatGPT) сгенерируйте **10 тестовых записей** вида:

```
ФИО, Модель_устройства, Стоимость_ремонта
```

---

### Требования к алгоритму

1.  Форма вызывается **в цикле 10 раз**.
2.  На каждой итерации вводится один набор данных.
3.  После нажатия ОК данные добавляются в файл `repair_requests.txt`.
4.  Каждая запись сохраняется с новой строки в формате:

   ```
   ФИО;Модель;Стоимость
   ```
5.  При нажатии Отмена ввод прекращается.
6.  После завершения вывести сообщение:

   ```
   Принято заявок на ремонт: N
   ```

# 🔵 Задание 2 (повышенный уровень)

### Постановка задачи

Создайте форму **«Расширенная заявка на ремонт техники»** с дополнительными элементами, условиями показа и использованием кнопки типа **Триггер**.

---

### Элементы формы

1.  Текст — ФИО владельца (`owner_fullname`)
2.  Текст — Модель устройства (`device_model`)
3.  Число — Стоимость ремонта (`repair_cost`)
4.  Чекбокс — Действует гарантия (`is_warranty`)
5.  Текст — Номер гарантийного талона (`warranty_number`)

   *   отображать только если `is_warranty = Истина`
6.  Селект — Тип неисправности (`failure_type`)

   *   варианты: hardware, software, mechanical
7.  Кнопки:

   *   Триггер **«Проверить ремонт»**
   *   ОК
   *   Отмена

---

### Логика кнопки Триггер

Создайте функцию `validate_repair_request(form_result)`, которая:

1.  Проверяет:

   *   `device_model` содержит хотя бы одну цифру
   *   И (`repair_cost` <= 30000 ИЛИ `is_warranty` = Истина)
2.  Формирует текст:

   *   «Ремонт возможен к выполнению»
   *   или «Ошибка: неверная модель или стоимость превышает лимит без гарантии»

Результатом работы функции является вывод сообщения на экран

⚠ Форма не должна закрываться при нажатии Триггера.

---

### Динамические параметры при вызове формы

Программно задать:

*   заголовок окна: **Сервисный центр 🔧**
*   изменить цвет фона;
*   значение по умолчанию: `failure_type = hardware`.

---

### Подготовка данных

Сгенерируйте с помощью ИИ **10 расширенных записей**:

```
ФИО, Модель, Стоимость, Гарантия (да/нет), Номер_талона, Тип_неисправности
```

---

### Требования к алгоритму

1.  Ввод выполняется последовательно для 10 заявок.
2.  Перед подтверждением обязательно нажимать кнопку «Проверить ремонт».
3.  После нажатия ОК данные сохраняются в файл `repair_requests_extended.csv`:

```
ФИО;Модель;Стоимость;Гарантия;Номер_талона;Тип_неисправности
```

4.  Если форма возвращает пустое значение (Отмена), выполнение прекращается.
