# ✅ Вариант 31

## 🟢 Задание 1 (базовый уровень)

### Постановка задачи

Создайте пользовательскую форму **«Регистрация пользователя мобильного приложения»** для последовательного ввода данных нескольких новых участников платформы.

### Элементы формы

Добавьте:

1.  Текст (строка) — Имя пользователя (`app_username`)
2.  Текст (строка) — ID устройства (`device_id`)
3.  Число — Версия ОС (`os_version`, min = 8, max = 17)
4.  Кнопки:

   *   Подтвердить (ОК)
   *   Отмена

---

### Подготовка данных

С помощью любого ИИ-сервиса (DeepSeek / ChatGPT) сгенерируйте **10 тестовых записей** вида:

```
Имя_пользователя, ID_устройства, Версия_ОС
```

---

### Требования к алгоритму

1.  Форма вызывается **в цикле 10 раз**.
2.  На каждой итерации вводится один набор данных.
3.  После нажатия ОК данные добавляются в файл `app_users.txt`.
4.  Каждая запись сохраняется с новой строки в формате:

   ```
   Имя_пользователя;ID_устройства;Версия_ОС
   ```
5.  При нажатии Отмена ввод прекращается.
6.  После завершения вывести сообщение:

   ```
   Зарегистрировано пользователей: N
   ```

# 🔵 Задание 2 (повышенный уровень)

### Постановка задачи

Создайте форму **«Расширенная регистрация в приложении»** с дополнительными элементами, условиями показа и использованием кнопки типа **Триггер**.

---

### Элементы формы

1.  Текст — Имя пользователя (`app_username`)
2.  Текст — ID устройства (`device_id`)
3.  Число — Версия ОС (`os_version`)
4.  Чекбокс — Включить push-уведомления (`enable_push`)
5.  Текст — Токен для уведомлений (`push_token`)

   *   отображать только если `enable_push = Истина`
6.  Селект — Тип подписки (`subscription_tier`)

   *   варианты: free, premium, ultimate
7.  Кнопки:

   *   Триггер **«Активировать аккаунт»**
   *   ОК
   *   Отмена

---

### Логика кнопки Триггер

Создайте функцию `activate_app_account(form_result)`, которая:

1.  Проверяет:

   *   (Длина ID устройства == 16 И ID состоит из букв и цифр)
   *   И (Версия ОС >= 10 ИЛИ (Включить push = Истина И длина токена == 32))
2.  Формирует текст:

   *   «Аккаунт активирован»
   *   или «Ошибка: неверный ID устройства или версия ОС/токен не соответствуют требованиям»

Результатом работы функции является вывод сообщения на экран

⚠ Форма не должна закрываться при нажатии Триггера.

---

### Динамические параметры при вызове формы

Программно задать:

*   заголовок окна: **Mobile App Register 📱**
*   изменить цвет фона;
*   значение по умолчанию: `subscription_tier = free`.

---

### Подготовка данных

Сгенерируйте с помощью ИИ **10 расширенных записей**:

```
Имя_пользователя, ID_устройства, Версия_ОС, Push (да/нет), Токен, Тип_подписки
```

---

### Требования к алгоритму

1.  Ввод выполняется последовательно для 10 пользователей.
2.  Перед подтверждением обязательно нажимать кнопку «Активировать аккаунт».
3.  После нажатия ОК данные сохраняются в файл `app_users_extended.csv`:

```
Имя_пользователя;ID_устройства;Версия_ОС;Push;Токен;Тип_подписки
```

4.  Если форма возвращает пустое значение (Отмена), выполнение прекращается.
