# ✅ Вариант 33

## 🟢 Задание 1 (базовый уровень)

### Постановка задачи

Создайте пользовательскую форму **«Регистрация клиента салона красоты»** для последовательного ввода данных нескольких посетителей, желающих записаться на процедуру.

### Элементы формы

Добавьте:

1.  Текст (строка) — ФИО клиента (`client_fullname`)
2.  Текст (строка) — Номер телефона для связи (`contact_phone`)
3.  Число — Длительность процедуры в минутах (`procedure_duration`, min = 15, max = 240)
4.  Кнопки:

   *   Подтвердить (ОК)
   *   Отмена

---

### Подготовка данных

С помощью любого ИИ-сервиса (DeepSeek / ChatGPT) сгенерируйте **10 тестовых записей** вида:

```
ФИО, Телефон, Длительность_мин
```

---

### Требования к алгоритму

1.  Форма вызывается **в цикле 10 раз**.
2.  На каждой итерации вводится один набор данных.
3.  После нажатия ОК данные добавляются в файл `beauty_clients.txt`.
4.  Каждая запись сохраняется с новой строки в формате:

   ```
   ФИО;Телефон;Длительность_мин
   ```
5.  При нажатии Отмена ввод прекращается.
6.  После завершения вывести сообщение:

   ```
   Зарегистрировано клиентов: N
   ```

# 🔵 Задание 2 (повышенный уровень)

### Постановка задачи

Создайте форму **«Расширенная запись в салон красоты»** с дополнительными элементами, условиями показа и использованием кнопки типа **Триггер**.

---

### Элементы формы

1.  Текст — ФИО клиента (`client_fullname`)
2.  Текст — Номер телефона для связи (`contact_phone`)
3.  Число — Длительность процедуры в минутах (`procedure_duration`)
4.  Чекбокс — Первая запись в салон (`is_first_visit`)
5.  Текст — Источник рекомендации (`referral_source`)

   *   отображать только если `is_first_visit = Истина`
6.  Селект — Тип услуги (`service_category`)

   *   варианты: hair, nails, spa, makeup
7.  Кнопки:

   *   Триггер **«Проверить запись»**
   *   ОК
   *   Отмена

---

### Логика кнопки Триггер

Создайте функцию `validate_appointment(form_result)`, которая:

1.  Проверяет:

   *   (Длина телефона >= 11 И телефон содержит только цифры и символы `+`, `-`)
   *   И (Длительность процедуры <= 180 ИЛИ (Первая запись = Истина И длина источника рекомендации >= 3))
2.  Формирует текст:

   *   «Запись подтверждена»
   *   или «Ошибка: неверный формат телефона или длительность/источник не соответствуют требованиям»

Результатом работы функции является вывод сообщения на экран

⚠ Форма не должна закрываться при нажатии Триггера.

---

### Динамические параметры при вызове формы

Программно задать:

*   заголовок окна: **Beauty Salon 💅**
*   изменить цвет фона;
*   значение по умолчанию: `service_category = hair`.

---

### Подготовка данных

Сгенерируйте с помощью ИИ **10 расширенных записей**:

```
ФИО, Телефон, Длительность_мин, Первый_визит (да/нет), Источник, Тип_услуги
```

---

### Требования к алгоритму

1.  Ввод выполняется последовательно для 10 клиентов.
2.  Перед подтверждением обязательно нажимать кнопку «Проверить запись».
3.  После нажатия ОК данные сохраняются в файл `beauty_clients_extended.csv`:

```
ФИО;Телефон;Длительность_мин;Первый_визит;Источник;Тип_услуги
```

4.  Если форма возвращает пустое значение (Отмена), выполнение прекращается.
