# ✅ Вариант 19

## 🟢 Задание 1 (базовый уровень)

### Постановка задачи

Создайте пользовательскую форму **«Регистрация волонтёра мероприятия»** для последовательного ввода данных нескольких добровольцев, желающих помочь в организации события.

### Элементы формы

Добавьте:

1.  Текст (строка) — ФИО волонтёра (`volunteer_name`)
2.  Текст (строка) — Контактный телефон (`contact_phone`)
3.  Число — Количество доступных часов (`available_hours`, min = 2, max = 40)
4.  Кнопки:

   *   Подтвердить (ОК)
   *   Отмена

---

### Подготовка данных

С помощью любого ИИ-сервиса (DeepSeek / ChatGPT) сгенерируйте **10 тестовых записей** вида:

```
ФИО, Телефон, Часы_доступности
```

---

### Требования к алгоритму

1.  Форма вызывается **в цикле 10 раз**.
2.  На каждой итерации вводится один набор данных.
3.  После нажатия ОК данные добавляются в файл `event_volunteers.txt`.
4.  Каждая запись сохраняется с новой строки в формате:

   ```
   ФИО;Телефон;Часы_доступности
   ```
5.  При нажатии Отмена ввод прекращается.
6.  После завершения вывести сообщение:

   ```
   Зарегистрировано волонтёров: N
   ```

# 🔵 Задание 2 (повышенный уровень)

### Постановка задачи

Создайте форму **«Расширенная регистрация волонтёра»** с дополнительными элементами, условиями показа и использованием кнопки типа **Триггер**.

---

### Элементы формы

1.  Текст — ФИО волонтёра (`volunteer_name`)
2.  Текст — Контактный телефон (`contact_phone`)
3.  Число — Количество доступных часов (`available_hours`)
4.  Чекбокс — Есть опыт волонтёрства (`has_volunteer_exp`)
5.  Текст — Название прошлого мероприятия (`prev_event_name`)

   *   отображать только если `has_volunteer_exp = Истина`
6.  Селект — Зона ответственности (`responsibility_zone`)

   *   варианты: registration, logistics, info, media
7.  Кнопки:

   *   Триггер **«Одобрить кандидата»**
   *   ОК
   *   Отмена

---

### Логика кнопки Триггер

Создайте функцию `approve_volunteer(form_result)`, которая:

1.  Проверяет:

   *   (Количество часов >= 4 И телефон начинается с `+7` или `8`)
   *   ИЛИ (Есть опыт = Истина И Зона ответственности != "media")
2.  Формирует текст:

   *   «Кандидат одобрен»
   *   или «Ошибка: мало часов/неверный телефон или опыт не подходит для выбранной зоны»

Результатом работы функции является вывод сообщения на экран

⚠ Форма не должна закрываться при нажатии Триггера.

---

### Динамические параметры при вызове формы

Программно задать:

*   заголовок окна: **Volunteer Hub 🤝**
*   изменить цвет фона;
*   значение по умолчанию: `responsibility_zone = registration`.

---

### Подготовка данных

Сгенерируйте с помощью ИИ **10 расширенных записей**:

```
ФИО, Телефон, Часы_доступности, Опыт (да/нет), Прошлое_мероприятие, Зона
```

---

### Требования к алгоритму

1.  Ввод выполняется последовательно для 10 волонтёров.
2.  Перед подтверждением обязательно нажимать кнопку «Одобрить кандидата».
3.  После нажатия ОК данные сохраняются в файл `event_volunteers_extended.csv`:

```
ФИО;Телефон;Часы_доступности;Опыт;Прошлое_мероприятие;Зона
```

4.  Если форма возвращает пустое значение (Отмена), выполнение прекращается.
