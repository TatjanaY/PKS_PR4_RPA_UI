# ✅ Вариант 8

## 🟢 Задание 1 (базовый уровень)

### Постановка задачи

Создайте пользовательскую форму **«Бронирование экскурсии»** для последовательного ввода данных нескольких туристов.

### Элементы формы

Добавьте:

1.  Текст (строка) — ФИО туриста (`tourist_fullname`)
2.  Текст (строка) — Email для подтверждения (`booking_email`)
3.  Число — Количество человек в группе (`group_size`, min = 1, max = 50)
4.  Кнопки:

   *   Подтвердить (ОК)
   *   Отмена

---

### Подготовка данных

С помощью любого ИИ-сервиса (DeepSeek / ChatGPT) сгенерируйте **10 тестовых записей** вида:

```
ФИО, Email, Количество_человек
```

---

### Требования к алгоритму

1.  Форма вызывается **в цикле 10 раз**.
2.  На каждой итерации вводится один набор данных.
3.  После нажатия ОК данные добавляются в файл `excursion_bookings.txt`.
4.  Каждая запись сохраняется с новой строки в формате:

   ```
   ФИО;Email;Количество
   ```
5.  При нажатии Отмена ввод прекращается.
6.  После завершения вывести сообщение:

   ```
   Оформлено бронирований: N
   ```

# 🔵 Задание 2 (повышенный уровень)

### Постановка задачи

Создайте форму **«Расширенное бронирование экскурсии»** с дополнительными элементами, условиями показа и использованием кнопки типа **Триггер**.

---

### Элементы формы

1.  Текст — ФИО туриста (`tourist_fullname`)
2.  Текст — Email для подтверждения (`booking_email`)
3.  Число — Количество человек (`group_size`)
4.  Чекбокс — Нужен трансфер (`need_transfer`)
5.  Текст — Адрес подачи (`pickup_address`)

   *   отображать только если `need_transfer = Истина`
6.  Селект — Тип экскурсии (`tour_type`)

   *   варианты: walking, bus, boat
7.  Кнопки:

   *   Триггер **«Проверить бронь»**
   *   ОК
   *   Отмена

---

### Логика кнопки Триггер

Создайте функцию `validate_booking(form_result)`, которая:

1.  Проверяет:

   *   `booking_email` заканчивается на `.ru` ИЛИ `.com`
   *   И (`group_size` <= 20 ИЛИ `need_transfer` = Ложь)
2.  Формирует текст:

   *   «Бронирование подтверждено»
   *   или «Ошибка: неверный email или группа слишком большая для трансфера»

Результатом работы функции является вывод сообщения на экран

⚠ Форма не должна закрываться при нажатии Триггера.

---

### Динамические параметры при вызове формы

Программно задать:

*   заголовок окна: **Экскурсии Travel✈️**
*   изменить цвет фона;
*   значение по умолчанию: `tour_type = walking`.

---

### Подготовка данных

Сгенерируйте с помощью ИИ **10 расширенных записей**:

```
ФИО, Email, Количество, Трансфер (да/нет), Адрес_подачи, Тип_экскурсии
```

---

### Требования к алгоритму

1.  Ввод выполняется последовательно для 10 туристов.
2.  Перед подтверждением обязательно нажимать кнопку «Проверить бронь».
3.  После нажатия ОК данные сохраняются в файл `excursion_bookings_extended.csv`:

```
ФИО;Email;Количество;Трансфер;Адрес_подачи;Тип_экскурсии
```

4.  Если форма возвращает пустое значение (Отмена), выполнение прекращается.
