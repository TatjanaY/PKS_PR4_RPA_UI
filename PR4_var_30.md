# ✅ Вариант 30

## 🟢 Задание 1 (базовый уровень)

### Постановка задачи

Создайте пользовательскую форму **«Заявка в библиотеку»** для последовательного ввода данных нескольких читателей.

### Элементы формы

Добавьте:

1.  Текст (строка) — ФИО читателя (`reader_fullname`)
2.  Текст (строка) — Номер читательского билета (`library_card_number`)
3.  Число — Количество книг для выдачи (`books_count`, min = 1, max = 10)
4.  Кнопки:

   *   Подтвердить (ОК)
   *   Отмена

---

### Подготовка данных

С помощью любого ИИ-сервиса (DeepSeek / ChatGPT) сгенерируйте **10 тестовых записей** вида:

```
ФИО, Номер_билета, Количество_книг
```

---

### Требования к алгоритму

1.  Форма вызывается **в цикле 10 раз**.
2.  На каждой итерации вводится один набор данных.
3.  После нажатия ОК данные добавляются в файл `library_requests.txt`.
4.  Каждая запись сохраняется с новой строки в формате:

   ```
   ФИО;Номер_билета;Количество_книг
   ```
5.  При нажатии Отмена ввод прекращается.
6.  После завершения вывести сообщение:

   ```
   Обработано заявок: N
   ```

# 🔵 Задание 2 (повышенный уровень)

### Постановка задачи

Создайте форму **«Расширенная заявка в библиотеку»** с дополнительными элементами, условиями показа и использованием кнопки типа **Триггер**.

---

### Элементы формы

1.  Текст — ФИО читателя (`reader_fullname`)
2.  Текст — Номер читательского билета (`library_card_number`)
3.  Число — Количество книг для выдачи (`books_count`)
4.  Чекбокс — Требуется продление срока (`needs_extension`)
5.  Текст — Причина продления (`extension_reason`)

   *   отображать только если `needs_extension = Истина`
6.  Селект — Категория книг (`book_category`)

   *   варианты: fiction, science, history
7.  Кнопки:

   *   Триггер **«Проверить заявку»**
   *   ОК
   *   Отмена

---

### Логика кнопки Триггер

Создайте функцию `validate_library_request(form_result)`, которая:

1.  Проверяет:

   *   `library_card_number` начинается с буквы `L` ИЛИ состоит из 6 цифр
   *   И (`books_count` <= 5 ИЛИ `needs_extension` = Ложь)
2.  Формирует текст:

   *   «Заявка принята в обработку»
   *   или «Ошибка: неверный номер билета или превышен лимит книг»

Результатом работы функции является вывод сообщения на экран

⚠ Форма не должна закрываться при нажатии Триггера.

---

### Динамические параметры при вызове формы

Программно задать:

*   заголовок окна: **Библиотека 📚**
*   изменить цвет фона;
*   значение по умолчанию: `book_category = fiction`.

---

### Подготовка данных

Сгенерируйте с помощью ИИ **10 расширенных записей**:

```
ФИО, Номер_билета, Количество_книг, Продление (да/нет), Причина_продления, Категория
```

---

### Требования к алгоритму

1.  Ввод выполняется последовательно для 10 читателей.
2.  Перед подтверждением обязательно нажимать кнопку «Проверить заявку».
3.  После нажатия ОК данные сохраняются в файл `library_requests_extended.csv`:

```
ФИО;Номер_билета;Количество_книг;Продление;Причина_продления;Категория
```

4.  Если форма возвращает пустое значение (Отмена), выполнение прекращается.
