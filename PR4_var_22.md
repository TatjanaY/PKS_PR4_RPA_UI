# ✅ Вариант 22

## 🟢 Задание 1 (базовый уровень)

### Постановка задачи

Создайте пользовательскую форму **«Заявка на место в коворкинге»** для последовательного ввода данных нескольких обращений фрилансеров.

### Элементы формы

Добавьте:

1.  Текст (строка) — ФИО заявителя (`applicant_name`)
2.  Текст (строка) — Email для уведомлений (`notification_email`)
3.  Число — Количество дней аренды (`rental_days`, min = 1, max = 30)
4.  Кнопки:

   *   Подтвердить (ОК)
   *   Отмена

---

### Подготовка данных

С помощью любого ИИ-сервиса (DeepSeek / ChatGPT) сгенерируйте **10 тестовых записей** вида:

```
ФИО, Email, Количество_дней
```

---

### Требования к алгоритму

1.  Форма вызывается **в цикле 10 раз**.
2.  На каждой итерации вводится один набор данных.
3.  После нажатия ОК данные добавляются в файл `coworking_requests.txt`.
4.  Каждая запись сохраняется с новой строки в формате:

   ```
   ФИО;Email;Дни
   ```
5.  При нажатии Отмена ввод прекращается.
6.  После завершения вывести сообщение:

   ```
   Обработано заявок: N
   ```

# 🔵 Задание 2 (повышенный уровень)

### Постановка задачи

Создайте форму **«Расширенная заявка в коворкинг»** с дополнительными элементами, условиями показа и использованием кнопки типа **Триггер**.

---

### Элементы формы

1.  Текст — ФИО заявителя (`applicant_name`)
2.  Текст — Email для уведомлений (`notification_email`)
3.  Число — Количество дней аренды (`rental_days`)
4.  Чекбокс — Нужен доступ 24/7 (`access_247`)
5.  Текст — Код пропуска (`access_code`)

   *   отображать только если `access_247 = Истина`
6.  Селект — Тип рабочего места (`workspace_type`)

   *   варианты: open, private, meeting
7.  Кнопки:

   *   Триггер **«Проверить доступность»**
   *   ОК
   *   Отмена

---

### Логика кнопки Триггер

Создайте функцию `validate_coworking_booking(form_result)`, которая:

1.  Проверяет:

   *   `notification_email` содержит домен `@mail.ru` ИЛИ `@bk.ru`
   *   И (`rental_days` <= 14 ИЛИ `access_247` = Истина)
2.  Формирует текст:

   *   «Место доступно для бронирования»
   *   или «Ошибка: неверный email или срок аренды превышает лимит»

Результатом работы функции является вывод сообщения на экран

⚠ Форма не должна закрываться при нажатии Триггера.

---

### Динамические параметры при вызове формы

Программно задать:

*   заголовок окна: **Коворкинг Space 🚀**
*   изменить цвет фона;
*   значение по умолчанию: `workspace_type = open`.

---

### Подготовка данных

Сгенерируйте с помощью ИИ **10 расширенных записей**:

```
ФИО, Email, Дни, Доступ_24/7 (да/нет), Код_пропуска, Тип_места
```

---

### Требования к алгоритму

1.  Ввод выполняется последовательно для 10 заявок.
2.  Перед подтверждением обязательно нажимать кнопку «Проверить доступность».
3.  После нажатия ОК данные сохраняются в файл `coworking_requests_extended.csv`:

```
ФИО;Email;Дни;Доступ_24/7;Код_пропуска;Тип_места
```

4.  Если форма возвращает пустое значение (Отмена), выполнение прекращается.
