# ✅ Вариант 24

## 🟢 Задание 1 (базовый уровень)

### Постановка задачи

Создайте пользовательскую форму **«Заявка на потребительский кредит»** для последовательного ввода данных нескольких обращений клиентов.

### Элементы формы

Добавьте:

1.  Текст (строка) — ФИО заёмщика (`borrower_fullname`)
2.  Текст (строка) — Номер паспорта (`passport_number`)
3.  Число — Сумма кредита в рублях (`loan_amount`, min = 10000, max = 3000000)
4.  Кнопки:

   *   Подтвердить (ОК)
   *   Отмена

---

### Подготовка данных

С помощью любого ИИ-сервиса (DeepSeek / ChatGPT) сгенерируйте **10 тестовых записей** вида:

```
ФИО, Номер_паспорта, Сумма_кредита
```

---

### Требования к алгоритму

1.  Форма вызывается **в цикле 10 раз**.
2.  На каждой итерации вводится один набор данных.
3.  После нажатия ОК данные добавляются в файл `credit_applications.txt`.
4.  Каждая запись сохраняется с новой строки в формате:

   ```
   ФИО;Паспорт;Сумма
   ```
5.  При нажатии Отмена ввод прекращается.
6.  После завершения вывести сообщение:

   ```
   Принято заявок на кредит: N
   ```

# 🔵 Задание 2 (повышенный уровень)

### Постановка задачи

Создайте форму **«Расширенная заявка на кредит»** с дополнительными элементами, условиями показа и использованием кнопки типа **Триггер**.

---

### Элементы формы

1.  Текст — ФИО заёмщика (`borrower_fullname`)
2.  Текст — Номер паспорта (`passport_number`)
3.  Число — Сумма кредита (`loan_amount`)
4.  Чекбокс — Есть поручитель (`has_guarantor`)
5.  Текст — ФИО поручителя (`guarantor_name`)

   *   отображать только если `has_guarantor = Истина`
6.  Селект — Срок кредита (`loan_term`)

   *   варианты: year1, year3, year5
7.  Кнопки:

   *   Триггер **«Проверить кредит»**
   *   ОК
   *   Отмена

---

### Логика кнопки Триггер

Создайте функцию `validate_credit_application(form_result)`, которая:

1.  Проверяет:

   *   `passport_number` состоит ровно из 10 цифр
   *   И (`loan_amount` <= 1500000 ИЛИ `has_guarantor` = Истина)
2.  Формирует текст:

   *   «Заявка одобрена предварительной проверкой»
   *   или «Ошибка: неверный паспорт или сумма превышает лимит без поручителя»

Результатом работы функции является вывод сообщения на экран

⚠ Форма не должна закрываться при нажатии Триггера.

---

### Динамические параметры при вызове формы

Программно задать:

*   заголовок окна: **Банк КредитОнлайн 💳**
*   изменить цвет фона;
*   значение по умолчанию: `loan_term = year3`.

---

### Подготовка данных

Сгенерируйте с помощью ИИ **10 расширенных записей**:

```
ФИО, Паспорт, Сумма, Поручитель (да/нет), ФИО_поручителя, Срок
```

---

### Требования к алгоритму

1.  Ввод выполняется последовательно для 10 заявок.
2.  Перед подтверждением обязательно нажимать кнопку «Проверить кредит».
3.  После нажатия ОК данные сохраняются в файл `credit_applications_extended.csv`:

```
ФИО;Паспорт;Сумма;Поручитель;ФИО_поручителя;Срок
```

4.  Если форма возвращает пустое значение (Отмена), выполнение прекращается.
