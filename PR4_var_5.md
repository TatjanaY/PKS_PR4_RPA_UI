# ✅ Вариант 5

## 🟢 Задание 1 (базовый уровень)

### Постановка задачи

Создайте пользовательскую форму **«Регистрация участника марафона»** для последовательного ввода данных нескольких бегунов.

### Элементы формы

Добавьте:

1.  Текст (строка) — ФИО (`runner_name`)
2.  Текст (строка) — Номер bib (`bib_number`)
3.  Число — Планируемое время в минутах (`plan_time`, min = 10, max = 600)
4.  Кнопки:

   *   Подтвердить (ОК)
   *   Отмена

---

### Подготовка данных

С помощью любого ИИ-сервиса (DeepSeek / ChatGPT) сгенерируйте **10 тестовых записей** вида:

```
ФИО, Номер_bib, Время_мин
```

---

### Требования к алгоритму

1.  Форма вызывается **в цикле 10 раз**.
2.  На каждой итерации вводится один набор данных.
3.  После нажатия ОК данные добавляются в файл `marathon_runners.txt`.
4.  Каждая запись сохраняется с новой строки в формате:

   ```
   ФИО;Номер_bib;Время_мин
   ```
5.  При нажатии Отмена ввод прекращается.
6.  После завершения вывести сообщение:

   ```
   Зарегистрировано бегунов: N
   ```

# 🔵 Задание 2 (повышенный уровень)

### Постановка задачи

Создайте форму **«Расширенная регистрация марафона»** с дополнительными элементами, условиями показа и использованием кнопки типа **Триггер**.

---

### Элементы формы

1.  Текст — ФИО (`runner_name`)
2.  Текст — Номер bib (`bib_number`)
3.  Число — Возраст (`age`)
4.  Чекбокс — Требуется мед. поддержка (`needs_medical`)
5.  Текст — Группа крови (`blood_type`)

   *   отображать только если `needs_medical = Истина`
6.  Селект — Дистанция (`distance_cat`)

   *   варианты: 5km, 10km, 21km, 42km
7.  Кнопки:

   *   Триггер **«Проверить допуск»**
   *   ОК
   *   Отмена

---

### Логика кнопки Триггер

Создайте функцию `check_eligibility(form_result)`, которая:

1.  Проверяет:

   *   (Длина номера bib >= 3 И номер состоит из цифр)
   *   И (Возраст >= 18 ИЛИ (Возраст >= 16 И Дистанция == "5km"))
2.  Формирует текст:

   *   «Участник допущен»
   *   или «Ошибка: неверный номер или возраст не подходит для дистанции»

Результатом работы функции является вывод сообщения на экран

⚠ Форма не должна закрываться при нажатии Триггера.

---

### Динамические параметры при вызове формы

Программно задать:

*   заголовок окна: **Marathon 2024 🏃**
*   изменить цвет фона;
*   значение по умолчанию: `distance_cat = 10km`.

---

### Подготовка данных

Сгенерируйте с помощью ИИ **10 расширенных записей**:

```
ФИО, Номер_bib, Возраст, Мед_поддержка (да/нет), Группа_крови, Дистанция
```

---

### Требования к алгоритму

1.  Ввод выполняется последовательно для 10 участников.
2.  Перед подтверждением обязательно нажимать кнопку «Проверить допуск».
3.  После нажатия ОК данные сохраняются в файл `marathon_runners_extended.csv`:

```
ФИО;Номер_bib;Возраст;Мед_поддержка;Группа_крови;Дистанция
```

4.  Если форма возвращает пустое значение (Отмена), выполнение прекращается.
