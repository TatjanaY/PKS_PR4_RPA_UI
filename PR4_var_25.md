# ✅ Вариант 25

## 🟢 Задание 1 (базовый уровень)

### Постановка задачи

Создайте пользовательскую форму **«Регистрация участника спортивного турнира»** для последовательного ввода данных нескольких спортсменов, желающих принять участие в соревновании.

### Элементы формы

Добавьте:

1.  Текст (строка) — ФИО спортсмена (`athlete_name`)
2.  Текст (строка) — Название команды (`team_name`)
3.  Число — Игровой номер (`player_number`, min = 1, max = 99)
4.  Кнопки:

   *   Подтвердить (ОК)
   *   Отмена

---

### Подготовка данных

С помощью любого ИИ-сервиса (DeepSeek / ChatGPT) сгенерируйте **10 тестовых записей** вида:

```
ФИО, Команда, Номер_игрока
```

---

### Требования к алгоритму

1.  Форма вызывается **в цикле 10 раз**.
2.  На каждой итерации вводится один набор данных.
3.  После нажатия ОК данные добавляются в файл `tournament_athletes.txt`.
4.  Каждая запись сохраняется с новой строки в формате:

   ```
   ФИО;Команда;Номер_игрока
   ```
5.  При нажатии Отмена ввод прекращается.
6.  После завершения вывести сообщение:

   ```
   Зарегистрировано спортсменов: N
   ```

# 🔵 Задание 2 (повышенный уровень)

### Постановка задачи

Создайте форму **«Расширенная регистрация турнира»** с дополнительными элементами, условиями показа и использованием кнопки типа **Триггер**.

---

### Элементы формы

1.  Текст — ФИО спортсмена (`athlete_name`)
2.  Текст — Название команды (`team_name`)
3.  Число — Игровой номер (`player_number`)
4.  Чекбокс — Капитан команды (`is_captain`)
5.  Текст — Контакт тренера (`coach_contact`)

   *   отображать только если `is_captain = Истина`
6.  Селект — Весовая категория (`weight_category`)

   *   варианты: light, medium, heavy, unlimited
7.  Кнопки:

   *   Триггер **«Проверить допуск»**
   *   ОК
   *   Отмена

---

### Логика кнопки Триггер

Создайте функцию `check_tournament_eligibility(form_result)`, которая:

1.  Проверяет:

   *   (Игровой номер > 0 И Игровой номер <= 99)
   *   И (Длина названия команды >= 4 ИЛИ (Капитан = Истина И длина контакта тренера >= 11))
2.  Формирует текст:

   *   «Участник допущен к турниру»
   *   или «Ошибка: неверный номер игрока или данные команды/тренера»

Результатом работы функции является вывод сообщения на экран

⚠ Форма не должна закрываться при нажатии Триггера.

---

### Динамические параметры при вызове формы

Программно задать:

*   заголовок окна: **Sports Tournament 🏆**
*   изменить цвет фона;
*   значение по умолчанию: `weight_category = medium`.

---

### Подготовка данных

Сгенерируйте с помощью ИИ **10 расширенных записей**:

```
ФИО, Команда, Номер_игрока, Капитан (да/нет), Контакт_тренера, Весовая_категория
```

---

### Требования к алгоритму

1.  Ввод выполняется последовательно для 10 спортсменов.
2.  Перед подтверждением обязательно нажимать кнопку «Проверить допуск».
3.  После нажатия ОК данные сохраняются в файл `tournament_athletes_extended.csv`:

```
ФИО;Команда;Номер_игрока;Капитан;Контакт_тренера;Весовая_категория
```

4.  Если форма возвращает пустое значение (Отмена), выполнение прекращается.
