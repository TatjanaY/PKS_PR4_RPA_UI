# ✅ Вариант 20

## 🟢 Задание 1 (базовый уровень)

### Постановка задачи

Создайте пользовательскую форму **«Учёт участника мастер-класса»** для последовательного ввода данных нескольких посетителей.

### Элементы формы

Добавьте:

1.  Текст (строка) — ФИО участника (`participant_fullname`)
2.  Текст (строка) — Контактный email (`contact_email`)
3.  Число — Уровень подготовки (1–10) (`skill_level`, min = 1, max = 10)
4.  Кнопки:

   *   Подтвердить (ОК)
   *   Отмена

---

### Подготовка данных

С помощью любого ИИ-сервиса (DeepSeek / ChatGPT) сгенерируйте **10 тестовых записей** вида:

```
ФИО, Email, Уровень_подготовки
```

---

### Требования к алгоритму

1.  Форма вызывается **в цикле 10 раз**.
2.  На каждой итерации вводится один набор данных.
3.  После нажатия ОК данные добавляются в файл `workshop_participants.txt`.
4.  Каждая запись сохраняется с новой строки в формате:

   ```
   ФИО;Email;Уровень
   ```
5.  При нажатии Отмена ввод прекращается.
6.  После завершения вывести сообщение:

   ```
   Зарегистрировано участников: N
   ```

# 🔵 Задание 2 (повышенный уровень)

### Постановка задачи

Создайте форму **«Расширенный учёт участника мастер-класса»** с дополнительными элементами, условиями показа и использованием кнопки типа **Триггер**.

---

### Элементы формы

1.  Текст — ФИО участника (`participant_fullname`)
2.  Текст — Контактный email (`contact_email`)
3.  Число — Уровень подготовки (`skill_level`)
4.  Чекбокс — Нужны материалы (`needs_materials`)
5.  Текст — Список материалов (`materials_list`)

   *   отображать только если `needs_materials = Истина`
6.  Селект — Формат участия (`participation_format`)

   *   варианты: offline, online, hybrid
7.  Кнопки:

   *   Триггер **«Проверить регистрацию»**
   *   ОК
   *   Отмена

---

### Логика кнопки Триггер

Создайте функцию `validate_workshop_registration(form_result)`, которая:

1.  Проверяет:

   *   `contact_email` содержит символ `@` И не пустой
   *   И (`skill_level` >= 3 ИЛИ `needs_materials` = Истина)
2.  Формирует текст:

   *   «Регистрация успешна»
   *   или «Ошибка: неверный email или низкий уровень без материалов»

Результатом работы функции является вывод сообщения на экран

⚠ Форма не должна закрываться при нажатии Триггера.

---

### Динамические параметры при вызове формы

Программно задать:

*   заголовок окна: **Мастер-класс Craft 🎨**
*   изменить цвет фона;
*   значение по умолчанию: `participation_format = offline`.

---

### Подготовка данных

Сгенерируйте с помощью ИИ **10 расширенных записей**:

```
ФИО, Email, Уровень, Материалы (да/нет), Список_материалов, Формат
```

---

### Требования к алгоритму

1.  Ввод выполняется последовательно для 10 участников.
2.  Перед подтверждением обязательно нажимать кнопку «Проверить регистрацию».
3.  После нажатия ОК данные сохраняются в файл `workshop_participants_extended.csv`:

```
ФИО;Email;Уровень;Материалы;Список_материалов;Формат
```

4.  Если форма возвращает пустое значение (Отмена), выполнение прекращается.
