# ✅ Вариант 12

## 🟢 Задание 1 (базовый уровень)

### Постановка задачи

Создайте пользовательскую форму **«Заявка на консультацию»** для последовательного ввода данных нескольких обращений клиентов.

### Элементы формы

Добавьте:

1.  Текст (строка) — ФИО клиента (`client_name`)
2.  Текст (строка) — Контактный телефон (`client_phone`)
3.  Число — Длительность консультации в минутах (`consult_duration`, min = 15, max = 120)
4.  Кнопки:

   *   Подтвердить (ОК)
   *   Отмена

---

### Подготовка данных

С помощью любого ИИ-сервиса (DeepSeek / ChatGPT) сгенерируйте **10 тестовых записей** вида:

```
ФИО, Телефон, Длительность
```

---

### Требования к алгоритму

1.  Форма вызывается **в цикле 10 раз**.
2.  На каждой итерации вводится один набор данных.
3.  После нажатия ОК данные добавляются в файл `consult_requests.txt`.
4.  Каждая запись сохраняется с новой строки в формате:

   ```
   ФИО;Телефон;Длительность
   ```
5.  При нажатии Отмена ввод прекращается.
6.  После завершения вывести сообщение:

   ```
   Принято заявок: N
   ```

# 🔵 Задание 2 (повышенный уровень)

### Постановка задачи

Создайте форму **«Расширенная заявка на консультацию»** с дополнительными элементами, условиями показа и использованием кнопки типа **Триггер**.

---

### Элементы формы

1.  Текст — ФИО клиента (`client_name`)
2.  Текст — Контактный телефон (`client_phone`)
3.  Число — Длительность консультации (`consult_duration`)
4.  Чекбокс — Требуется эксперт (`need_expert`)
5.  Текст — Специализация эксперта (`expert_specialization`)

   *   отображать только если `need_expert = Истина`
6.  Селект — Формат консультации (`consult_format`)

   *   варианты: online, offline, phone
7.  Кнопки:

   *   Триггер **«Проверить заявку»**
   *   ОК
   *   Отмена

---

### Логика кнопки Триггер

Создайте функцию `validate_consult_request(form_result)`, которая:

1.  Проверяет:

   *   `client_phone` начинается с `+7` ИЛИ `8`
   *   И (`consult_duration` % 15 == 0 ИЛИ `need_expert` = Истина)
2.  Формирует текст:

   *   «Заявка соответствует требованиям»
   *   или «Ошибка: неверный формат телефона или длительность не кратна 15»

Результатом работы функции является вывод сообщения на экран

⚠ Форма не должна закрываться при нажатии Триггера.

---

### Динамические параметры при вызове формы

Программно задать:

*   заголовок окна: **Консультации Pro 💼**
*   изменить цвет фона;
*   значение по умолчанию: `consult_format = online`.

---

### Подготовка данных

Сгенерируйте с помощью ИИ **10 расширенных записей**:

```
ФИО, Телефон, Длительность, Эксперт (да/нет), Специализация, Формат
```

---

### Требования к алгоритму

1.  Ввод выполняется последовательно для 10 заявок.
2.  Перед подтверждением обязательно нажимать кнопку «Проверить заявку».
3.  После нажатия ОК данные сохраняются в файл `consult_requests_extended.csv`:

```
ФИО;Телефон;Длительность;Эксперт;Специализация;Формат
```

4.  Если форма возвращает пустое значение (Отмена), выполнение прекращается.
