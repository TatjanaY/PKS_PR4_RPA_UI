# ✅ Вариант 4

## 🟢 Задание 1 (базовый уровень)

### Постановка задачи

Создайте пользовательскую форму **«Заявка в техподдержку»** для последовательного ввода данных нескольких обращений.

### Элементы формы

Добавьте:

1.  Текст (строка) — Имя заявителя (`requester_name`)
2.  Текст (строка) — Контактные данные (`contact_detail`)
3.  Число — Срочность (`urgency_level`, min = 1, max = 10)
4.  Кнопки:

   *   Подтвердить (ОК)
   *   Отмена

---

### Подготовка данных

С помощью любого ИИ-сервиса (DeepSeek / ChatGPT) сгенерируйте **10 тестовых записей** вида:

```
Имя, Контакты, Срочность
```

---

### Требования к алгоритму

1.  Форма вызывается **в цикле 10 раз**.
2.  На каждой итерации вводится один набор данных.
3.  После нажатия ОК данные добавляются в файл `support_tickets.txt`.
4.  Каждая запись сохраняется с новой строки в формате:

   ```
   Имя;Контакты;Срочность
   ```
5.  При нажатии Отмена ввод прекращается.
6.  После завершения вывести сообщение:

   ```
   Обработано заявок: N
   ```

# 🔵 Задание 2 (повышенный уровень)

### Постановка задачи

Создайте форму **«Расширенная заявка в техподдержку»** с дополнительными элементами, условиями показа и использованием кнопки типа **Триггер**.

---

### Элементы формы

1.  Текст — Имя заявителя (`requester_name`)
2.  Текст — Контактные данные (`contact_detail`)
3.  Число — Срочность (`urgency_level`)
4.  Чекбокс — Есть договор SLA (`has_sla`)
5.  Текст — Номер договора (`sla_code`)

   *   отображать только если `has_sla = Истина`
6.  Селект — Категория проблемы (`problem_category`)

   *   варианты: hardware, software, network
7.  Кнопки:

   *   Триггер **«Валидировать»**
   *   ОК
   *   Отмена

---

### Логика кнопки Триггер

Создайте функцию `validate_ticket_request(form_result)`, которая:

1.  Проверяет:

   *   `contact_detail` содержит символ `@` ИЛИ начинается с цифры `8`
   *   И (`urgency_level` >= 5 ИЛИ `has_sla` = Истина)
2.  Формирует текст:

   *   «Заявка принята в работу»
   *   или «Ошибка: неверные контакты или низкий приоритет»

Результатом работы функции является вывод сообщения на экран

⚠ Форма не должна закрываться при нажатии Триггера.

---

### Динамические параметры при вызове формы

Программно задать:

*   заголовок окна: **HelpDesk 🎧**
*   изменить цвет фона;
*   значение по умолчанию: `problem_category = software`.

---

### Подготовка данных

Сгенерируйте с помощью ИИ **10 расширенных записей**:

```
Имя, Контакты, Срочность, SLA (да/нет), Номер_договора, Категория
```

---

### Требования к алгоритму

1.  Ввод выполняется последовательно для 10 заявок.
2.  Перед подтверждением обязательно нажимать кнопку «Валидировать».
3.  После нажатия ОК данные сохраняются в файл `support_tickets_extended.csv`:

```
Имя;Контакты;Срочность;SLA;Номер_договора;Категория
```

4.  Если форма возвращает пустое значение (Отмена), выполнение прекращается.
