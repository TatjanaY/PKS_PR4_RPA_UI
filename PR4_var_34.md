# ✅ Вариант 34

## 🟢 Задание 1 (базовый уровень)

### Постановка задачи

Создайте пользовательскую форму **«Заявка на стажировку»** для последовательного ввода данных нескольких соискателей.

### Элементы формы

Добавьте:

1.  Текст (строка) — ФИО соискателя (`applicant_fullname`)
2.  Текст (строка) — Контактный email (`contact_email`)
3.  Число — Курс обучения (`study_course`, min = 1, max = 6)
4.  Кнопки:

   *   Подтвердить (ОК)
   *   Отмена

---

### Подготовка данных

С помощью любого ИИ-сервиса (DeepSeek / ChatGPT) сгенерируйте **10 тестовых записей** вида:

```
ФИО, Email, Курс
```

---

### Требования к алгоритму

1.  Форма вызывается **в цикле 10 раз**.
2.  На каждой итерации вводится один набор данных.
3.  После нажатия ОК данные добавляются в файл `internship_applications.txt`.
4.  Каждая запись сохраняется с новой строки в формате:

   ```
   ФИО;Email;Курс
   ```
5.  При нажатии Отмена ввод прекращается.
6.  После завершения вывести сообщение:

   ```
   Принято заявок на стажировку: N
   ```

# 🔵 Задание 2 (повышенный уровень)

### Постановка задачи

Создайте форму **«Расширенная заявка на стажировку»** с дополнительными элементами, условиями показа и использованием кнопки типа **Триггер**.

---

### Элементы формы

1.  Текст — ФИО соискателя (`applicant_fullname`)
2.  Текст — Контактный email (`contact_email`)
3.  Число — Курс обучения (`study_course`)
4.  Чекбокс — Есть рекомендательное письмо (`has_recommendation`)
5.  Текст — ФИО рекомендателя (`recommender_name`)

   *   отображать только если `has_recommendation = Истина`
6.  Селект — Направление стажировки (`internship_direction`)

   *   варианты: it, marketing, hr
7.  Кнопки:

   *   Триггер **«Проверить заявку»**
   *   ОК
   *   Отмена

---

### Логика кнопки Триггер

Создайте функцию `validate_internship_application(form_result)`, которая:

1.  Проверяет:

   *   `contact_email` содержит домен `@edu.ru` ИЛИ `@student.ru`
   *   И (`study_course` >= 3 ИЛИ `has_recommendation` = Истина)
2.  Формирует текст:

   *   «Заявка соответствует требованиям стажировки»
   *   или «Ошибка: неверный email или ранний курс без рекомендации»

Результатом работы функции является вывод сообщения на экран

⚠ Форма не должна закрываться при нажатии Триггера.

---

### Динамические параметры при вызове формы

Программно задать:

*   заголовок окна: **Стажировка Start🎯**
*   изменить цвет фона;
*   значение по умолчанию: `internship_direction = it`.

---

### Подготовка данных

Сгенерируйте с помощью ИИ **10 расширенных записей**:

```
ФИО, Email, Курс, Рекомендация (да/нет), ФИО_рекомендателя, Направление
```

---

### Требования к алгоритму

1.  Ввод выполняется последовательно для 10 соискателей.
2.  Перед подтверждением обязательно нажимать кнопку «Проверить заявку».
3.  После нажатия ОК данные сохраняются в файл `internship_applications_extended.csv`:

```
ФИО;Email;Курс;Рекомендация;ФИО_рекомендателя;Направление
```

4.  Если форма возвращает пустое значение (Отмена), выполнение прекращается.
