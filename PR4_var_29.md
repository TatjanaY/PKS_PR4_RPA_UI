# ✅ Вариант 29

## 🟢 Задание 1 (базовый уровень)

### Постановка задачи

Создайте пользовательскую форму **«Регистрация сотрудника на корпоративное обучение»** для последовательного ввода данных нескольких работников, желающих пройти повышение квалификации.

### Элементы формы

Добавьте:

1.  Текст (строка) — ФИО сотрудника (`employee_name`)
2.  Текст (строка) — Табельный номер (`employee_id`)
3.  Число — Стаж работы в компании (лет) (`work_experience`, min = 0, max = 40)
4.  Кнопки:

   *   Подтвердить (ОК)
   *   Отмена

---

### Подготовка данных

С помощью любого ИИ-сервиса (DeepSeek / ChatGPT) сгенерируйте **10 тестовых записей** вида:

```
ФИО, Табельный_номер, Стаж_лет
```

---

### Требования к алгоритму

1.  Форма вызывается **в цикле 10 раз**.
2.  На каждой итерации вводится один набор данных.
3.  После нажатия ОК данные добавляются в файл `corporate_training.txt`.
4.  Каждая запись сохраняется с новой строки в формате:

   ```
   ФИО;Табельный_номер;Стаж_лет
   ```
5.  При нажатии Отмена ввод прекращается.
6.  После завершения вывести сообщение:

   ```
   Зарегистрировано сотрудников: N
   ```

# 🔵 Задание 2 (повышенный уровень)

### Постановка задачи

Создайте форму **«Расширенная регистрация на обучение»** с дополнительными элементами, условиями показа и использованием кнопки типа **Триггер**.

---

### Элементы формы

1.  Текст — ФИО сотрудника (`employee_name`)
2.  Текст — Табельный номер (`employee_id`)
3.  Число — Стаж работы в компании (лет) (`work_experience`)
4.  Чекбокс — Обучение за счёт компании (`company_funded`)
5.  Текст — Код бюджета отдела (`budget_code`)

   *   отображать только если `company_funded = Истина`
6.  Селект — Направление обучения (`training_track`)

   *   варианты: management, technical, soft_skills, compliance
7.  Кнопки:

   *   Триггер **«Проверить заявку»**
   *   ОК
   *   Отмена

---

### Логика кнопки Триггер

Создайте функцию `validate_training_application(form_result)`, которая:

1.  Проверяет:

   *   (Длина табельного номера == 6 И номер начинается с "EMP")
   *   И (Стаж >= 1 ИЛИ (Обучение за счёт компании = Истина И длина кода бюджета == 8))
2.  Формирует текст:

   *   «Заявка одобрена»
   *   или «Ошибка: неверный табельный номер или стаж/бюджет не соответствуют требованиям»

Результатом работы функции является вывод сообщения на экран

⚠ Форма не должна закрываться при нажатии Триггера.

---

### Динамические параметры при вызове формы

Программно задать:

*   заголовок окна: **Corporate Learning 📚**
*   изменить цвет фона;
*   значение по умолчанию: `training_track = technical`.

---

### Подготовка данных

Сгенерируйте с помощью ИИ **10 расширенных записей**:

```
ФИО, Табельный_номер, Стаж_лет, Оплата_компанией (да/нет), Код_бюджета, Направление
```

---

### Требования к алгоритму

1.  Ввод выполняется последовательно для 10 сотрудников.
2.  Перед подтверждением обязательно нажимать кнопку «Проверить заявку».
3.  После нажатия ОК данные сохраняются в файл `corporate_training_extended.csv`:

```
ФИО;Табельный_номер;Стаж_лет;Оплата_компанией;Код_бюджета;Направление
```

4.  Если форма возвращает пустое значение (Отмена), выполнение прекращается.
