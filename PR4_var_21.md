# ✅ Вариант 21

## 🟢 Задание 1 (базовый уровень)

### Постановка задачи

Создайте пользовательскую форму **«Регистрация посетителя выставки»** для последовательного ввода данных нескольких гостей культурного мероприятия.

### Элементы формы

Добавьте:

1.  Текст (строка) — ФИО посетителя (`visitor_fullname`)
2.  Текст (строка) — Номер билета (`ticket_id`)
3.  Число — Возраст посетителя (`visitor_age`, min = 0, max = 120)
4.  Кнопки:

   *   Подтвердить (ОК)
   *   Отмена

---

### Подготовка данных

С помощью любого ИИ-сервиса (DeepSeek / ChatGPT) сгенерируйте **10 тестовых записей** вида:

```
ФИО, Номер_билета, Возраст
```

---

### Требования к алгоритму

1.  Форма вызывается **в цикле 10 раз**.
2.  На каждой итерации вводится один набор данных.
3.  После нажатия ОК данные добавляются в файл `exhibition_visitors.txt`.
4.  Каждая запись сохраняется с новой строки в формате:

   ```
   ФИО;Номер_билета;Возраст
   ```
5.  При нажатии Отмена ввод прекращается.
6.  После завершения вывести сообщение:

   ```
   Зарегистрировано посетителей: N
   ```

# 🔵 Задание 2 (повышенный уровень)

### Постановка задачи

Создайте форму **«Расширенная регистрация на выставку»** с дополнительными элементами, условиями показа и использованием кнопки типа **Триггер**.

---

### Элементы формы

1.  Текст — ФИО посетителя (`visitor_fullname`)
2.  Текст — Номер билета (`ticket_id`)
3.  Число — Возраст посетителя (`visitor_age`)
4.  Чекбокс — Льготная категория (`is_beneficiary`)
5.  Текст — Номер удостоверения льготы (`benefit_doc_num`)

   *   отображать только если `is_beneficiary = Истина`
6.  Селект — Тип экспозиции (`exhibition_type`)

   *   варианты: art, history, science, photo
7.  Кнопки:

   *   Триггер **«Проверить пропуск»**
   *   ОК
   *   Отмена

---

### Логика кнопки Триггер

Создайте функцию `validate_exhibition_pass(form_result)`, которая:

1.  Проверяет:

   *   (Длина номера билета == 8 И номер состоит из букв и цифр)
   *   И (Возраст < 18 ИЛИ Возраст >= 60 ИЛИ Льготная категория = Ложь)
2.  Формирует текст:

   *   «Пропуск активен»
   *   или «Ошибка: неверный формат билета или возраст не соответствует льготной категории»

Результатом работы функции является вывод сообщения на экран

⚠ Форма не должна закрываться при нажатии Триггера.

---

### Динамические параметры при вызове формы

Программно задать:

*   заголовок окна: **Exhibition Gallery 🎨**
*   изменить цвет фона;
*   значение по умолчанию: `exhibition_type = art`.

---

### Подготовка данных

Сгенерируйте с помощью ИИ **10 расширенных записей**:

```
ФИО, Номер_билета, Возраст, Льготник (да/нет), Номер_удостоверения, Тип_экспозиции
```

---

### Требования к алгоритму

1.  Ввод выполняется последовательно для 10 посетителей.
2.  Перед подтверждением обязательно нажимать кнопку «Проверить пропуск».
3.  После нажатия ОК данные сохраняются в файл `exhibition_visitors_extended.csv`:

```
ФИО;Номер_билета;Возраст;Льготник;Номер_удостоверения;Тип_экспозиции
```

4.  Если форма возвращает пустое значение (Отмена), выполнение прекращается.
