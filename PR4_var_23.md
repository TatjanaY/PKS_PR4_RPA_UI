# ✅ Вариант 23

## 🟢 Задание 1 (базовый уровень)

### Постановка задачи

Создайте пользовательскую форму **«Регистрация клиента страховой компании»** для последовательного ввода данных нескольких страхователей.

### Элементы формы

Добавьте:

1.  Текст (строка) — ФИО клиента (`client_fullname`)
2.  Текст (строка) — Номер договора (`contract_number`)
3.  Число — Сумма страхования в тыс. руб. (`insurance_amount`, min = 10, max = 10000)
4.  Кнопки:

   *   Подтвердить (ОК)
   *   Отмена

---

### Подготовка данных

С помощью любого ИИ-сервиса (DeepSeek / ChatGPT) сгенерируйте **10 тестовых записей** вида:

```
ФИО, Номер_договора, Сумма_тыс_руб
```

---

### Требования к алгоритму

1.  Форма вызывается **в цикле 10 раз**.
2.  На каждой итерации вводится один набор данных.
3.  После нажатия ОК данные добавляются в файл `insurance_clients.txt`.
4.  Каждая запись сохраняется с новой строки в формате:

   ```
   ФИО;Номер_договора;Сумма_тыс_руб
   ```
5.  При нажатии Отмена ввод прекращается.
6.  После завершения вывести сообщение:

   ```
   Зарегистрировано клиентов: N
   ```

# 🔵 Задание 2 (повышенный уровень)

### Постановка задачи

Создайте форму **«Расширенная регистрация страхового клиента»** с дополнительными элементами, условиями показа и использованием кнопки типа **Триггер**.

---

### Элементы формы

1.  Текст — ФИО клиента (`client_fullname`)
2.  Текст — Номер договора (`contract_number`)
3.  Число — Сумма страхования в тыс. руб. (`insurance_amount`)
4.  Чекбокс — Расширенное покрытие (`is_extended_coverage`)
5.  Текст — Код дополнительного риска (`risk_code`)

   *   отображать только если `is_extended_coverage = Истина`
6.  Селект — Тип страхования (`insurance_type`)

   *   варианты: life, property, health, auto
7.  Кнопки:

   *   Триггер **«Верифицировать полис»**
   *   ОК
   *   Отмена

---

### Логика кнопки Триггер

Создайте функцию `verify_insurance_policy(form_result)`, которая:

1.  Проверяет:

   *   (Длина номера договора == 12 И номер начинается с "INS")
   *   И (Сумма страхования >= 100 ИЛИ (Расширенное покрытие = Истина И длина кода риска == 6))
2.  Формирует текст:

   *   «Полис верифицирован»
   *   или «Ошибка: неверный формат договора или сумма/код риска не соответствуют требованиям»

Результатом работы функции является вывод сообщения на экран

⚠ Форма не должна закрываться при нажатии Триггера.

---

### Динамические параметры при вызове формы

Программно задать:

*   заголовок окна: **Insurance Portal 🛡️**
*   изменить цвет фона;
*   значение по умолчанию: `insurance_type = property`.

---

### Подготовка данных

Сгенерируйте с помощью ИИ **10 расширенных записей**:

```
ФИО, Номер_договора, Сумма_тыс_руб, Расширенное_покрытие (да/нет), Код_риска, Тип_страхования
```

---

### Требования к алгоритму

1.  Ввод выполняется последовательно для 10 клиентов.
2.  Перед подтверждением обязательно нажимать кнопку «Верифицировать полис».
3.  После нажатия ОК данные сохраняются в файл `insurance_clients_extended.csv`:

```
ФИО;Номер_договора;Сумма_тыс_руб;Расширенное_покрытие;Код_риска;Тип_страхования
```

4.  Если форма возвращает пустое значение (Отмена), выполнение прекращается.
