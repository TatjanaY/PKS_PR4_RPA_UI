# ✅ Вариант 2

## 🟢 Задание 1 (базовый уровень)

### Постановка задачи

Создайте пользовательскую форму **«Запись пациента»** для последовательного ввода данных нескольких посетителей клиники.

### Элементы формы

Добавьте:

1.  Текст (строка) — ФИО пациента (`client_fullname`)
2.  Текст (строка) — Мобильный телефон (`mobile_number`)
3.  Число — Возраст (`patient_age`, min = 0, max = 120)
4.  Кнопки:

   *   Подтвердить (ОК)
   *   Отмена

---

### Подготовка данных

С помощью любого ИИ-сервиса (DeepSeek / ChatGPT) сгенерируйте **10 тестовых записей** вида:

```
ФИО, Телефон, Возраст
```

---

### Требования к алгоритму

1.  Форма вызывается **в цикле 10 раз**.
2.  На каждой итерации вводится один набор данных.
3.  После нажатия ОК данные добавляются в файл `patients.txt`.
4.  Каждая запись сохраняется с новой строки в формате:

   ```
   ФИО;Телефон;Возраст
   ```
5.  При нажатии Отмена ввод прекращается.
6.  После завершения вывести сообщение:

   ```
   Зарегистрировано пациентов: N
   ```

# 🔵 Задание 2 (повышенный уровень)

### Постановка задачи

Создайте форму **«Расширенная запись в клинику»** с дополнительными элементами, условиями показа и использованием кнопки типа **Триггер**.

---

### Элементы формы

1.  Текст — ФИО пациента (`client_fullname`)
2.  Текст — Мобильный телефон (`mobile_number`)
3.  Число — Возраст (`patient_age`)
4.  Чекбокс — Есть полис ДМС (`is_insured`)
5.  Текст — Номер полиса (`insurance_code`)

   *   отображать только если `is_insured = Истина`
6.  Селект — Направление к врачу (`department_type`)

   *   варианты: therapist, surgeon, pediatrician
7.  Кнопки:

   *   Триггер **«Проверить запись»**
   *   ОК
   *   Отмена

---

### Логика кнопки Триггер

Создайте функцию `validate_appointment(form_result)`, которая:

1.  Проверяет:

   *   длина `mobile_number` равна 11 символам
   *   И (возраст `patient_age` ≥ 14 ИЛИ наличие полиса `is_insured` = Истина)
2.  Формирует текст:

   *   «Запись возможна»
   *   или «Ошибка: неверный телефон или возраст без полиса»

Результатом работы функции является вывод сообщения на экран

⚠ Форма не должна закрываться при нажатии Триггера.

---

### Динамические параметры при вызове формы

Программно задать:

*   заголовок окна: **Клиника Здоровья 🏥**
*   изменить цвет фона;
*   значение по умолчанию: `department_type = therapist`.

---

### Подготовка данных

Сгенерируйте с помощью ИИ **10 расширенных записей**:

```
ФИО, Телефон, Возраст, Полис (да/нет), Номер_полиса, Врач
```

---

### Требования к алгоритму

1.  Ввод выполняется последовательно для 10 пациентов.
2.  Перед подтверждением обязательно нажимать кнопку «Проверить запись».
3.  После нажатия ОК данные сохраняются в файл `patients_extended.csv`:

```
ФИО;Телефон;Возраст;Полис;Номер_полиса;Врач
```

4.  Если форма возвращает пустое значение (Отмена), выполнение прекращается.
