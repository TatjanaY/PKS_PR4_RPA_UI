# ✅ Вариант 15

## 🟢 Задание 1 (базовый уровень)

### Постановка задачи

Создайте пользовательскую форму **«Регистрация студента факультатива»** для последовательного ввода данных нескольких обучающихся, желающих посещать дополнительный курс.

### Элементы формы

Добавьте:

1.  Текст (строка) — ФИО студента (`student_name`)
2.  Текст (строка) — Номер зачётной книжки (`record_book_num`)
3.  Число — Курс обучения (`study_year`, min = 1, max = 6)
4.  Кнопки:

   *   Подтвердить (ОК)
   *   Отмена

---

### Подготовка данных

С помощью любого ИИ-сервиса (DeepSeek / ChatGPT) сгенерируйте **10 тестовых записей** вида:

```
ФИО, Номер_зачётки, Курс
```

---

### Требования к алгоритму

1.  Форма вызывается **в цикле 10 раз**.
2.  На каждой итерации вводится один набор данных.
3.  После нажатия ОК данные добавляются в файл `elective_students.txt`.
4.  Каждая запись сохраняется с новой строки в формате:

   ```
   ФИО;Номер_зачётки;Курс
   ```
5.  При нажатии Отмена ввод прекращается.
6.  После завершения вывести сообщение:

   ```
   Зарегистрировано студентов: N
   ```

# 🔵 Задание 2 (повышенный уровень)

### Постановка задачи

Создайте форму **«Расширенная регистрация на факультатив»** с дополнительными элементами, условиями показа и использованием кнопки типа **Триггер**.

---

### Элементы формы

1.  Текст — ФИО студента (`student_name`)
2.  Текст — Номер зачётной книжки (`record_book_num`)
3.  Число — Курс обучения (`study_year`)
4.  Чекбокс — Есть базовые знания (`has_prerequisites`)
5.  Текст — Название пройденного курса (`prev_course_name`)

   *   отображать только если `has_prerequisites = Истина`
6.  Селект — Направление факультатива (`elective_track`)

   *   варианты: analytics, design, programming
7.  Кнопки:

   *   Триггер **«Проверить зачисление»**
   *   ОК
   *   Отмена

---

### Логика кнопки Триггер

Создайте функцию `check_enrollment(form_result)`, которая:

1.  Проверяет:

   *   (Курс обучения >= 2 И длина номера зачётки == 6)
   *   ИЛИ (Есть базовые знания = Истина И длина названия курса >= 5)
2.  Формирует текст:

   *   «Студент зачислен»
   *   или «Ошибка: курс ниже 2 или неверные данные о подготовке»

Результатом работы функции является вывод сообщения на экран

⚠ Форма не должна закрываться при нажатии Триггера.

---

### Динамические параметры при вызове формы

Программно задать:

*   заголовок окна: **Elective Courses 🎓**
*   изменить цвет фона;
*   значение по умолчанию: `elective_track = programming`.

---

### Подготовка данных

Сгенерируйте с помощью ИИ **10 расширенных записей**:

```
ФИО, Номер_зачётки, Курс, Базовые_знания (да/нет), Пройденный_курс, Направление
```

---

### Требования к алгоритму

1.  Ввод выполняется последовательно для 10 студентов.
2.  Перед подтверждением обязательно нажимать кнопку «Проверить зачисление».
3.  После нажатия ОК данные сохраняются в файл `elective_students_extended.csv`:

```
ФИО;Номер_зачётки;Курс;Базовые_знания;Пройденный_курс;Направление
```

4.  Если форма возвращает пустое значение (Отмена), выполнение прекращается.
