# ✅ Вариант 11

## 🟢 Задание 1 (базовый уровень)

### Постановка задачи

Создайте пользовательскую форму **«Регистрация участника конференции»** для последовательного ввода данных нескольких докладчиков и слушателей.

### Элементы формы

Добавьте:

1.  Текст (строка) — ФИО участника (`participant_name`)
2.  Текст (строка) — Организация (`organization`)
3.  Число — Номер секции (`section_number`, min = 1, max = 10)
4.  Кнопки:

   *   Подтвердить (ОК)
   *   Отмена

---

### Подготовка данных

С помощью любого ИИ-сервиса (DeepSeek / ChatGPT) сгенерируйте **10 тестовых записей** вида:

```
ФИО, Организация, Номер_секции
```

---

### Требования к алгоритму

1.  Форма вызывается **в цикле 10 раз**.
2.  На каждой итерации вводится один набор данных.
3.  После нажатия ОК данные добавляются в файл `conference_participants.txt`.
4.  Каждая запись сохраняется с новой строки в формате:

   ```
   ФИО;Организация;Номер_секции
   ```
5.  При нажатии Отмена ввод прекращается.
6.  После завершения вывести сообщение:

   ```
   Зарегистрировано участников: N
   ```

# 🔵 Задание 2 (повышенный уровень)

### Постановка задачи

Создайте форму **«Расширенная регистрация конференции»** с дополнительными элементами, условиями показа и использованием кнопки типа **Триггер**.

---

### Элементы формы

1.  Текст — ФИО участника (`participant_name`)
2.  Текст — Организация (`organization`)
3.  Число — Номер секции (`section_number`)
4.  Чекбокс — Докладчик с презентацией (`is_speaker`)
5.  Текст — Тема доклада (`presentation_topic`)

   *   отображать только если `is_speaker = Истина`
6.  Селект — Формат участия (`participation_format`)

   *   варианты: offline, online, hybrid
7.  Кнопки:

   *   Триггер **«Подтвердить участие»**
   *   ОК
   *   Отмена

---

### Логика кнопки Триггер

Создайте функцию `confirm_participation(form_result)`, которая:

1.  Проверяет:

   *   (Длина названия Организации >= 3 И Номер секции в диапазоне 1-10)
   *   И (Формат участия != "offline" ИЛИ (Формат == "offline" И длина Темы доклада >= 10))
2.  Формирует текст:

   *   «Участие подтверждено»
   *   или «Ошибка: неверные данные организации или тема доклада для офлайн-участия»

Результатом работы функции является вывод сообщения на экран

⚠ Форма не должна закрываться при нажатии Триггера.

---

### Динамические параметры при вызове формы

Программно задать:

*   заголовок окна: **Conference 2024 🎤**
*   изменить цвет фона;
*   значение по умолчанию: `participation_format = hybrid`.

---

### Подготовка данных

Сгенерируйте с помощью ИИ **10 расширенных записей**:

```
ФИО, Организация, Номер_секции, Докладчик (да/нет), Тема_доклада, Формат
```

---

### Требования к алгоритму

1.  Ввод выполняется последовательно для 10 участников.
2.  Перед подтверждением обязательно нажимать кнопку «Подтвердить участие».
3.  После нажатия ОК данные сохраняются в файл `conference_participants_extended.csv`:

```
ФИО;Организация;Номер_секции;Докладчик;Тема_доклада;Формат
```

4.  Если форма возвращает пустое значение (Отмена), выполнение прекращается.
