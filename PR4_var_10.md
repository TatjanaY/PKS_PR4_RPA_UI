# ✅ Вариант 10

## 🟢 Задание 1 (базовый уровень)

### Постановка задачи

Создайте пользовательскую форму **«Оформление заказа в интернет-магазине»** для последовательного ввода данных нескольких покупок.

### Элементы формы

Добавьте:

1.  Текст (строка) — ФИО покупателя (`buyer_name`)
2.  Текст (строка) — Email для чека (`receipt_email`)
3.  Число — Сумма заказа (`order_total`, min = 100, max = 100000)
4.  Кнопки:

   *   Подтвердить (ОК)
   *   Отмена

---

### Подготовка данных

С помощью любого ИИ-сервиса (DeepSeek / ChatGPT) сгенерируйте **10 тестовых записей** вида:

```
ФИО, Email, Сумма_заказа
```

---

### Требования к алгоритму

1.  Форма вызывается **в цикле 10 раз**.
2.  На каждой итерации вводится один набор данных.
3.  После нажатия ОК данные добавляются в файл `store_orders.txt`.
4.  Каждая запись сохраняется с новой строки в формате:

   ```
   ФИО;Email;Сумма
   ```
5.  При нажатии Отмена ввод прекращается.
6.  После завершения вывести сообщение:

   ```
   Оформлено заказов: N
   ```

# 🔵 Задание 2 (повышенный уровень)

### Постановка задачи

Создайте форму **«Расширенное оформление заказа»** с дополнительными элементами, условиями показа и использованием кнопки типа **Триггер**.

---

### Элементы формы

1.  Текст — ФИО покупателя (`buyer_name`)
2.  Текст — Email для чека (`receipt_email`)
3.  Число — Сумма заказа (`order_total`)
4.  Чекбокс — Подарочная упаковка (`is_gift_wrap`)
5.  Текст — Текст открытки (`gift_message`)

   *   отображать только если `is_gift_wrap = Истина`
6.  Селект — Способ доставки (`delivery_method`)

   *   варианты: courier, pickup, post
7.  Кнопки:

   *   Триггер **«Проверить заказ»**
   *   ОК
   *   Отмена

---

### Логика кнопки Триггер

Создайте функцию `validate_store_order(form_result)`, которая:

1.  Проверяет:

   *   `receipt_email` содержит символ `@` И длина строки > 5
   *   И (`order_total` >= 500 ИЛИ `is_gift_wrap` = Истина)
2.  Формирует текст:

   *   «Заказ принят к обработке»
   *   или «Ошибка: неверный email или сумма заказа слишком мала»

Результатом работы функции является вывод сообщения на экран

⚠ Форма не должна закрываться при нажатии Триггера.

---

### Динамические параметры при вызове формы

Программно задать:

*   заголовок окна: **Магазин OnlineShop 🛒**
*   изменить цвет фона;
*   значение по умолчанию: `delivery_method = courier`.

---

### Подготовка данных

Сгенерируйте с помощью ИИ **10 расширенных записей**:

```
ФИО, Email, Сумма, Подарок (да/нет), Текст_открытки, Доставка
```

---

### Требования к алгоритму

1.  Ввод выполняется последовательно для 10 заказов.
2.  Перед подтверждением обязательно нажимать кнопку «Проверить заказ».
3.  После нажатия ОК данные сохраняются в файл `store_orders_extended.csv`:

```
ФИО;Email;Сумма;Подарок;Текст_открытки;Доставка
```

4.  Если форма возвращает пустое значение (Отмена), выполнение прекращается.
