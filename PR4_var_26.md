# ✅ Вариант 26

## 🟢 Задание 1 (базовый уровень)

### Постановка задачи

Создайте пользовательскую форму **«Бронирование коворкинг-места»** для последовательного ввода данных нескольких клиентов.

### Элементы формы

Добавьте:

1.  Текст (строка) — ФИО клиента (`client_fullname`)
2.  Текст (строка) — Контактный телефон (`client_phone`)
3.  Число — Номер места (`seat_number`, min = 1, max = 100)
4.  Кнопки:

   *   Подтвердить (ОК)
   *   Отмена

---

### Подготовка данных

С помощью любого ИИ-сервиса (DeepSeek / ChatGPT) сгенерируйте **10 тестовых записей** вида:

```
ФИО, Телефон, Номер_места
```

---

### Требования к алгоритму

1.  Форма вызывается **в цикле 10 раз**.
2.  На каждой итерации вводится один набор данных.
3.  После нажатия ОК данные добавляются в файл `coworking_seats.txt`.
4.  Каждая запись сохраняется с новой строки в формате:

   ```
   ФИО;Телефон;Номер_места
   ```
5.  При нажатии Отмена ввод прекращается.
6.  После завершения вывести сообщение:

   ```
   Забронировано мест: N
   ```

# 🔵 Задание 2 (повышенный уровень)

### Постановка задачи

Создайте форму **«Расширенное бронирование коворкинга»** с дополнительными элементами, условиями показа и использованием кнопки типа **Триггер**.

---

### Элементы формы

1.  Текст — ФИО клиента (`client_fullname`)
2.  Текст — Контактный телефон (`client_phone`)
3.  Число — Номер места (`seat_number`)
4.  Чекбокс — Требуется проектор (`needs_projector`)
5.  Текст — ID оборудования (`equipment_id`)

   *   отображать только если `needs_projector = Истина`
6.  Селект — Период бронирования (`booking_period`)

   *   варианты: hour, day, week
7.  Кнопки:

   *   Триггер **«Проверить место»**
   *   ОК
   *   Отмена

---

### Логика кнопки Триггер

Создайте функцию `validate_seat_booking(form_result)`, которая:

1.  Проверяет:

   *   `client_phone` начинается с `+` ИЛИ содержит ровно 11 символов
   *   И (`seat_number` % 2 == 0 ИЛИ `needs_projector` = Истина)
2.  Формирует текст:

   *   «Место доступно для бронирования»
   *   или «Ошибка: неверный телефон или место не соответствует требованиям»

Результатом работы функции является вывод сообщения на экран

⚠ Форма не должна закрываться при нажатии Триггера.

---

### Динамические параметры при вызове формы

Программно задать:

*   заголовок окна: **Коворкинг Desk 🪑**
*   изменить цвет фона;
*   значение по умолчанию: `booking_period = day`.

---

### Подготовка данных

Сгенерируйте с помощью ИИ **10 расширенных записей**:

```
ФИО, Телефон, Номер_места, Проектор (да/нет), ID_оборудования, Период
```

---

### Требования к алгоритму

1.  Ввод выполняется последовательно для 10 клиентов.
2.  Перед подтверждением обязательно нажимать кнопку «Проверить место».
3.  После нажатия ОК данные сохраняются в файл `coworking_seats_extended.csv`:

```
ФИО;Телефон;Номер_места;Проектор;ID_оборудования;Период
```

4.  Если форма возвращает пустое значение (Отмена), выполнение прекращается.
